# Lumia ScriptorRAG 品牌命名体系

生成时间：2026-07-06  
依据：`C:\Users\35160\Desktop\lumia-base-rebrand.html` 品牌方案与当前文档智能体产品能力整理。

## 主产品名

**Lumia ScriptorRAG｜序光文枢 RAG**

- **Lumia**：沿用母品牌，表达“有光的解决路径”，强调温暖科技、长期陪伴与低成本 AI 工作流。
- **Scriptor**：来自拉丁语“书写者、抄写员”，也让人联想到中世纪整理典籍的 scriptorium；它对应本产品把扫描 PDF、古籍、资料文档转写成 Markdown 与知识库的能力。
- **RAG**：直接保留技术属性，说明产品不是普通 OCR 工具，而是面向“可检索、可引用、可问答”的文档知识系统。
- **文枢**：中文名取“文献中枢、知识枢纽”之意；既有古典气质，也能承载 OCR、排版、检索和问答的系统属性。

## 一句话定位

Lumia ScriptorRAG 是序光品牌下的文档智能体，把 PDF、扫描件与本地资料转化为可阅读、可检索、可追溯引用的知识库。

## 品牌短句

- 让旧纸重见微光，让知识有迹可循。
- 从纸面到文枢，从识别到引用。
- 把散落文档，整理成会回答的知识系统。
- 先识文，再成章，终入文枢。

## 产品栏目命名

| 旧栏目 | 新栏目 | 英文辅助名 | 含义 |
|---|---|---|---|
| OCR 智能体 | 光棱 OCR | Prism OCR | 像棱镜分光一样自动选择 Surya、MinerU、Docling 等最佳识别路径。 |
| 产品导览 | 序章导览 | Prologue | 作为用户进入产品世界观和操作流的第一章。 |
| 排版优化 | 字炉排版 | Glyph Forge | 将 OCR 草稿放入“字炉”重铸，清理噪声、修复结构、输出干净 Markdown。 |
| RAG 知识库 | 文枢 RAG | ScriptorRAG Vault | 文献汇入知识中枢，支持来源去重、古籍加权和可追溯问答。 |
| API 设置 | 星轨模型 | Model Orbit | 多家 LLM Provider 像星轨一样接入同一个智能工作流。 |

## 智能体命名建议

| 智能体 | 名称 | 职责 |
|---|---|---|
| OCR 路由智能体 | 光棱官 | 分析文档类型、选择 OCR 引擎、处理失败回退。 |
| 排版智能体 | 字炉匠 | 清洗 Markdown、修正标题/列表/段落/噪声字符。 |
| 检索智能体 | 文枢使 | 从知识库检索至少 5 条不同来源，并按相关度排序。 |
| 校正智能体 | 校雠官 | 结合 LLM 与规则修正表格、公式、顺序和明显 OCR 错误。 |
| 模型管理智能体 | 星轨司 | 管理 OpenAI、DeepSeek、GLM、Kimi、Qwen、Claude 等 Provider。 |

## 命名边界

- 母品牌保留 **Lumia｜序光**，文档产品命名为 **Lumia ScriptorRAG｜序光文枢 RAG**。
- 代码目录和历史环境变量可保留兼容旧名，避免破坏已有本地数据。
- 面向用户的 UI、文档、安装包、项目介绍和简历文案统一使用新名。
